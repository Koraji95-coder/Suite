# Suite Docs Index

This is the canonical entry point for repository documentation. People and tooling should start here, then move into the owning section README before drilling into individual notes.

## Getting Started

This section covers the minimum steps to get the project running locally. See the root [`README.md`](../README.md) for the full reference, and [`docs/runtime-control/workstation-bringup.md`](./runtime-control/workstation-bringup.md) for managed workstation setup.

### Prerequisites

- **Node.js + npm** — for the frontend and all repo tooling scripts
- **Python 3.14** (pinned in `.python-version`) — for the Flask backend
- **.NET SDK 8+** — for `Suite.RuntimeControl` and `suite-cad-authoring`
- **Windows workstation** — required for live AutoCAD flows and CAD-backed operations
- **AutoCAD** — required for any CAD-backed operations; not needed for pure frontend/backend work

### Install Dependencies

```bash
npm install
python -m pip install -r backend/requirements-api.lock.txt
dotnet restore dotnet/Suite.RuntimeControl/Suite.RuntimeControl.csproj
dotnet restore dotnet/suite-cad-authoring/SuiteCadAuthoring.csproj
dotnet restore dotnet/autodraft-api-contract/AutoDraft.ApiContract.csproj
```

> **Note:** `requirements-api.lock.txt` is generated by `pip-compile` and should not be edited manually. To update Python dependencies, modify `requirements-api.in` and regenerate the lock file with `pip-compile`.

### Environment Setup

Copy the example environment file and sync it:

```bash
copy .env.example .env
npm run env:sync:dry
npm run env:sync
```

At minimum, set these variables in `.env`:

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`
- `VITE_API_KEY`
- `API_KEY`

### Start the Native Dev Lane

> **Warning:** Never run `npm run dev:full` and `npm run workstation:bootstrap` at the same time on the same machine. Use one startup lane at a time.

For active frontend or backend coding, use the native dev lane:

```bash
npm run dev:full
```

This starts the frontend (Vite HMR), the Flask backend, ensures documentation and architecture artifacts are up to date, starts the AutoDraft contract service, and auto-starts the shared Redis service when needed. It does **not** start local Supabase automatically — run `npm run supabase:mode:local` separately if your work requires it.

For the managed workstation lane (used by Runtime Control and the Windows sign-in task), use `npm run workstation:bootstrap` instead.

### Validate Your Setup

Run the full validation sequence before your first commit:

```bash
npm run check        # lint, typecheck, guards, env parity, docs/arch artifacts
npm run test:unit    # Vitest unit tests
npm run build        # production build
```

For backend route changes also run:

```bash
npm run check:security:routes
```

See [`docs/development/ci-cd-pipeline.md`](./development/ci-cd-pipeline.md) for the full CI/CD pipeline reference.
