# ZeroClaw Adoption Matrix (Suite)

Historical record of what Suite extracted, referenced, or intentionally left behind from the removed `zeroclaw-main` subtree.

| Subsystem | Adopt into Suite | Reference only | Left behind when `zeroclaw-main` was removed | Evidence |
| --- | --- | --- | --- | --- |
| `web` | Logs viewer behavior patterns (pause/resume, filters, capped stream) | `agent-office.html`; web grouping/layout ideas only | Tailwind web app implementation | Suite uses CSS Modules/primitives shell (`src/routes/AppShell.tsx`, `src/routes/AppShell.module.css`) and enforces no Tailwind (`package.json` `guard:tailwind`). ZeroClaw web is Tailwind (`zeroclaw-main/web/src/index.css`, `zeroclaw-main/web/package.json`, `zeroclaw-main/web/src/pages/Logs.tsx`, `zeroclaw-main/web/src/components/layout/Layout.tsx`). |
| `doctor/health` | Doctor/preflight patterns; typed status/health contract ideas | N/A | N/A | Suite already has runtime doctor and health/status payloads (`src/lib/runtimeDoctor.ts`, `backend/route_groups/api_health.py`, `backend/route_groups/api_websocket_status.py`). ZeroClaw has doctor UX + typed API contracts (`zeroclaw-main/web/src/pages/Doctor.tsx`, `zeroclaw-main/web/src/types/api.ts`). |
| `config/schema` | Config masking/edit preservation patterns | N/A | N/A | ZeroClaw config masking + restore on save (`zeroclaw-main/src/gateway/api.rs`, `zeroclaw-main/web/src/pages/Config.tsx`, `zeroclaw-main/web/src/pages/Integrations.tsx`). Suite has runtime config normalization surface to align with (`backend/route_groups/api_runtime_config.py`). |
| `pairing/security` | N/A | Boundary/reference concepts only | ZeroClaw auth/pairing UI flow | Suite has canonical broker-first pairing and Supabase callback rules (`docs/agent/pairing-lifecycle.md`, `docs/security/auth-architecture-canonical.md`, `backend/route_groups/api_agent.py`, `src/routes/agent/AgentPairingCallbackPage.tsx`). ZeroClaw pairing/auth internals remain separate (`zeroclaw-main/src/security/pairing.rs`, `zeroclaw-main/src/gateway/api.rs`). |
| `gateway/api` | Typed status/health contract ideas; selective endpoint shape ideas | Compatibility endpoint patterns as reference | N/A | Suite gateway path is policy-locked (`docs/development/gateway-stability-policy.md`, `package.json` `gateway:dev`). | `docs/site tooling` | Docs manifest/inventory generation pipeline | N/A | N/A | ZeroClaw has concrete generator + manifest + inventory loop (`zeroclaw-main/site/scripts/generate-docs-manifest.mjs`, `zeroclaw-main/site/src/generated/docs-manifest.json`, `zeroclaw-main/docs/docs-inventory.md`). Suite docs are indexed but not manifest-generated (`docs/README.md`). |
| `runtime/service model` | N/A | N/A | CLI/service/runtime model | ZeroClaw runtime/service stack is Rust CLI + daemon/init/runtime adapters (`zeroclaw-main/src/service/mod.rs`, `zeroclaw-main/src/runtime/mod.rs`, `zeroclaw-main/src/main.rs`). Suite runtime is frontend + Python backend + scripts orchestration (`package.json`, `backend/api_server.py`, `scripts/run-agent-gateway.mjs`). |
| `channels/providers/memory/peripherals` | N/A | Specific implementation ideas only when explicitly requested | Generic provider/channel/memory platform; unrelated hardware/peripheral domains | ZeroClaw includes broad platform subsystems (`zeroclaw-main/src/channels/mod.rs`, `zeroclaw-main/src/providers/mod.rs`, `zeroclaw-main/src/memory/mod.rs`, `zeroclaw-main/src/peripherals/mod.rs`, `zeroclaw-main/firmware/*`). Suite scope is narrower and domain-specific (agent profile routing, local learning, AutoDraft/AutoWire boundaries per root guardrails). |

## Default decisions locked in this matrix

- Adopt: doctor/preflight patterns, logs viewer behavior, config masking/edit patterns, docs manifest/inventory generation, typed status/health contract ideas.
- Reference only: `agent-office.html`, web grouping/layout ideas.
- Left behind with the removed subtree: Tailwind web app, ZeroClaw auth/pairing UI flow, CLI/service/runtime model, generic provider/channel/memory platform, unrelated hardware/peripheral domains.
